# Praktische Prüfung

## Allgemeines

- Erledige diese Aufgaben in deinem Fork des sysadmin26 repositories.
- Erstelle für jede Aufgabe eine shell Datei im `code` Ordner, die den Kommandozeilen Code enthält, mit dem du die Aufgabe gelöst hast.
- Erstelle eine Extradatei `solutions.txt` in der du die Lösungen (zu 2.-4.) dokumentierst.
- Erstelle nach der Lösung jeder Aufgabe (mindestens) einen git commit.
- Wenn du alle Aufgaben erledigt hast, erstelle einen pull-request beim CCTB-UW/sysadmin26 repository. (Falls es dabei heißt, dass es Konflikte gibt, kannst du das ignorieren und den pull-request trotzdem erstellen).
- Schreibe Markus im Zoom Chat eine Nachricht, dass du fertig bist und warte auf seine Bestätigung, bevor du den Zoomraum verlässt.

## Aufgaben

1. Erstelle im `data/exam` Ordner einen Unterordner `index` und verschiebe alle `.fai` Dateien dort hinein. Erstelle einen git commit mit der geänderten Dateistruktur.
2. Wie viele Zeilen lang ist die Datei `data/exam/01.fa`?
3. Wie viele Sequenzen in `data/exam/02.fa` enthalten "bta" in der ID?
4. Was ist die kleinste und was ist die größte Zahl in `data/exam/numbers.txt`?
5. Lösche die Datei `data/exam/trash.txt`. Erstelle danach einen git commit.
